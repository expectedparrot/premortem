# Failure Reasons

Failure reasons should be rich narratives, not thin risk bullets.

Episodic reasons describe event chains: one thing happened, which caused another,
which produced a concrete failure.

Structural reasons describe underlying conditions that made the failure more
likely: incentives, governance, calendars, resource constraints, stakeholder
power, process mismatches, or market structure.

Strong reasons name concrete systems, dates, offices, budgets, timelines,
commitments, and stakeholders.

Typical command:

`premortem job generate reasons --domain "..." --good-example "..."`
